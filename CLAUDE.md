# CLAUDE.md

## HTML実装ルール

このリポジトリは LP サイトです。実装時は見た目だけでなく、アクセシビリティと SEO を意識し、セマンティックな HTML を優先してください。

## 基本方針

- `div` は最後の手段にし、意味に合う要素を先に選ぶ
- 見た目維持より、意味的に正しい HTML を優先する
- JavaScript が無効でも主要コンテンツは成立させる
- スクリーンリーダーとキーボード操作を前提に実装する

## 必須ルール

- 1ページに `main` は原則1つ
- 各ページの主見出しは `h1` を1つ
- 順不同の一覧は `ul/li`
- 手順やステップは `ol/li`
- 指標や項目名と値の組は `dl/dt/dd`
- 独立したカードは必要に応じて `article`
- FAQ や開閉 UI は `button` か `details/summary`
- `div role="button"` は原則使わない
- フォームの送信結果やエラーは `aria-live` または `role="status"` を使う
- 装飾アイコンは `aria-hidden="true"` を付ける
- 数値や本文を「初期HTMLでは誤った値にしてJSで後から直す」実装はしない
- canonical が必要なページは `link rel="canonical"` を出す

## 実装時チェック

- `main` は適切か
- `h1` と見出し階層は自然か
- 一覧が `div` の羅列になっていないか
- クリック要素が `button` / `a` になっているか
- 装飾アイコンが読み上げ対象になっていないか
- フォーム通知が支援技術に伝わるか
- JS 無効でも主要内容が読めるか

## 避けること

- レイアウト都合だけの `div` 多用
- 見た目調整のためだけの見出しタグ使用
- `div role="button"` の疑似UI
- 色だけで状態を伝える実装
- JS 依存で本文や数値が壊れる実装
